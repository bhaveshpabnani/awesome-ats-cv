# ATS Resume Resource Library

Use these resources to cross-check the workflow. Prefer university career centers and employer-neutral guidance when deciding baseline rules; use commercial ATS tools as secondary validation rather than absolute truth.

## University and Career-Center Sources

### MIT CAPD: Make Your Resume ATS-Friendly

- URL: https://capd.mit.edu/resources/make-your-resume-ats-friendly/
- Key ideas:
  - ATS systems parse resumes as text and may compare content to job postings
  - Boring formatting is safer for ATS submissions
  - Avoid images, graphics, tables, and text boxes
  - Use common fonts at 10pt or higher
  - Use job-posting keywords meaningfully and avoid vague filler
  - Save as plain text or extract text to check missing/wrong-order content

### Harvard Mignone Center: Create a Strong Resume

- URL: https://careerservices.fas.harvard.edu/resources/create-a-strong-resume/
- Key ideas:
  - Tailor the resume to the position
  - Keep language specific, active, concise, fact-based, and easy to scan
  - Quantify and qualify achievements
  - Avoid spelling/grammar errors, missing contact info, passive language, poor organization, and missing results
  - Keep formatting consistent and verify PDF conversion

### Columbia Career Education: Optimizing Your Resume for ATS

- URL: https://www.careereducation.columbia.edu/resources/optimizing-your-resume-applicant-tracking-systems
- Key ideas:
  - Use conventional headings that systems can categorize
  - Align resume language with the job posting
  - Keep the document easy for software and humans to interpret

### Santa Clara University Career Center: Common ATS Formatting Mistakes

- URL: https://www.scu.edu/careercenter/toolkit/job-scan-common-ats-resume-formatting-mistakes/
- Key ideas:
  - Keep date formats consistent
  - Use common headings
  - Avoid tables, headers/footers, graphics, and non-standard bullets
  - Use both long-form and acronym variants when keyword matching matters

### Yale Office of Career Strategy: Resume Action Verbs

- URL: https://ocs.yale.edu/resources/resume-action-verbs/
- Key ideas:
  - Use precise action verbs grouped by skill type
  - Select verbs that reflect ownership, analysis, leadership, research, communication, and technical contribution

## General Career Guidance

### Indeed: ATS Resume Template and Tips

- URL: https://www.indeed.com/career-advice/resumes-cover-letters/ats-resume-template
- Key ideas:
  - Use simple formatting and clear headings
  - Avoid headers, tables, graphics, images, and columns
  - Use standard fonts at 10-12pt
  - Use job-description keywords naturally in summary, skills, and experience
  - Include acronyms and full forms where relevant
  - Check requested file type before submitting

### The Muse: Action Verbs

- URL: https://www.themuse.com/advice/185-powerful-verbs-that-will-make-your-resume-awesome
- Key ideas:
  - Replace tired duty phrases with action verbs
  - Choose verbs based on the kind of impact: created, improved, led, researched, communicated, supported, or regulated

## ATS Tooling and Commercial References

### Jobscan: ATS Formatting Mistakes

- URL: https://www.jobscan.co/blog/ats-formatting-mistakes/
- Key ideas:
  - Avoid layout structures that interrupt text parsing
  - Use standard section headings
  - Keep contact information outside headers/footers
  - Tailor keywords to the job description

### Jobscan: ATS-Friendly Resume Templates

- URL: https://www.jobscan.co/blog/20-ats-friendly-resume-templates/
- Key ideas:
  - ATS-friendly templates are usually simple, reverse chronological, and text-first
  - Human readability and ATS readability should both be optimized

### Skillsyncer

- URL: https://skillsyncer.com/
- Key ideas:
  - Useful as a secondary job-description keyword coverage check
  - Do not accept every keyword suggestion blindly

### Resume Worded

- URL: https://resumeworded.com/
- Key ideas:
- Useful for high-level review of impact, brevity, and phrasing
- Validate suggestions against truth and target role relevance

### Grammarly

- URL: https://www.grammarly.com/
- Key ideas:
  - Useful as a final manual grammar, spelling, clarity, concision, and tone pass
  - Accept suggestions selectively because resume bullets often use compact fragments and technical proper nouns
  - Rerun layout and extraction validation after grammar edits

### LanguageTool

- URL: https://languagetool.org/http-api/
- Key ideas:
  - Useful for automated grammar checks through a public or self-hosted HTTP API
  - Prefer self-hosting for private resumes when possible
  - Treat automated suggestions as advisory

## Practical Resource Workflow

- Use MIT, Harvard, Columbia, Santa Clara, Yale, and Indeed as baseline rules
- Use The Muse/Yale for verb variety
- Use Jobscan/Skillsyncer/Resume Worded as optional second opinions
- Use Grammarly or LanguageTool for grammar/proofreading review
- Use this package's scripts as local, reproducible validation
- Treat every tool score as advisory; the final resume must remain truthful, targeted, and explainable
